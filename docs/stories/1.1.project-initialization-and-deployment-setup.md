# Story 1.1: 项目初始化与部署设置

## Status
Done

## Story
**As a** 项目所有者,
**I want** 一个初始化好的 Next.js Monorepo 项目，并将其配置好在 Vercel 上进行自动化部署,
**so that** 我们拥有一个稳固的、具备持续集成/持续部署（CI/CD）能力的技术基础和一个可供后续开发的实时网址。

## Acceptance Criteria
1.  一个新的 Monorepo 仓库已创建，其中包含一个使用 Next.js 和 TypeScript 的 Web 应用包。
2.  项目已连接到一个 Git 仓库（如 GitHub）。
3.  一个基础的欢迎页面或应用入口已成功部署到一个 Vercel 的公开 URL 上。
4.  Vercel 部署流程已配置为在代码推送到主分支时自动触发。

- [x] (AC: 1) 初始化一个空的 monorepo 仓库。
- [x] (AC: 1) 在 `apps/web` 目录下创建一个新的 Next.js 应用，使用 TypeScript。 [Source: docs/architecture/8-unified-project-structure.md]
- [x] (AC: 1) 在 `packages/shared-types` 目录下创建一个共享类型包。 [Source: docs/architecture/8-unified-project-structure.md]
- [x] (AC: 2) 将项目连接到 GitHub 仓库。

- [x] (AC: 4) 配置 Vercel 以在推送到 `main` 分支时自动部署。
- [x] (AC: 3) 确认部署成功，并且可以在 Vercel URL 上访问一个基础页面。
