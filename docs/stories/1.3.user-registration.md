# Story 1.3: 用户注册功能

## Status
Blocked

## Story
**As a** 新访客,
**I want** 使用我的邮箱和密码注册一个新账户,
**so that** 我可以开始使用这个应用。

## Acceptance Criteria
1.  应用提供一个包含邮箱和密码输入框的注册表单。
2.  成功提交后，将在 Supabase Auth 中创建一个新用户。
3.  用户在注册后会自动登录，并被重定向到应用主页。
4.  当输入的邮箱格式无效或该邮箱已被注册时，会向用户显示清晰的错误提示。

## Tasks / Subtasks
- [x] (AC: 1) 在 `apps/web/app/(auth)/register/page.tsx` 创建注册页面组件。 [Source: docs/architecture/8-unified-project-structure.md]
- [x] (AC: 1) 使用 Shadcn/ui 组件库构建一个包含邮箱、密码输入框和提交按钮的表单。 [Source: docs/architecture/3-tech-stack.md]
- [x] (AC: 2) 实现表单提交逻辑，调用 Supabase Auth 的 `signUp` 方法。 [Source: docs/architecture/3-tech-stack.md]
- [x] (AC: 3) 在成功注册后，使用 Next.js 的路由器将用户重定向到主页 (`/`)。
- [x] (AC: 4) 实现表单验证和错误处理逻辑，以显示来自 Supabase Auth 的错误信息。
- [x] (AC: 1, 4) 根据 `docs/architecture/10-testing-strategy.md` 的要求，为注册表单组件编写单元测试。

## Dev Notes
### 认证
- **Provider**: Supabase Auth [Source: docs/architecture/3-tech-stack.md]
- **方法**: 使用 Supabase JavaScript 客户端库 (`@supabase/supabase-js`) 中的 `supabase.auth.signUp()` 函数来处理用户注册。

### 项目结构
- **注册页面**: `apps/web/app/(auth)/register/page.tsx` [Source: docs/architecture/8-unified-project-structure.md]
- **UI 组件**: 应使用 `apps/web/components/ui/` 下的 Shadcn/ui 组件。 [Source: docs/architecture/8-unified-project-structure.md]
- **Supabase 客户端**: 应该从 `apps/web/lib/supabase/` 导入一个集中的 Supabase 客户端实例。 [Source: docs/architecture/8-unified-project-structure.md]

### API 规范
此故事不涉及创建新的后端 API 路由。所有与认证相关的操作都应通过前端直接与 Supabase Auth 服务交互来完成。 [Source: docs/architecture/5-api-specification.md]

### 编码标准
- **类型**: 如果需要定义任何特定于注册流程的数据结构，应在 `packages/shared-types` 中进行。 [Source: docs/architecture/11-coding-standards.md]

### 测试
- **策略**: 应对注册表单组件 (`register/page.tsx`) 进行单元测试。
- **范围**: 测试应覆盖：
    - 表单的成功渲染。
    - 用户输入。
    - 成功提交后的重定向行为（模拟）。
    - 显示错误消息的行为（模拟）。
- **工具**: Jest + React Testing Library (RTL) [Source: docs/architecture/10-testing-strategy.md]
